## 0001 - 비밀번호 해싱 알고리즘 선택

- 상태: 초안
- 날짜: 2025-11-12

### 배경

현재 사용자 회원가입 시 비밀번호를 해싱하기 위해 Spring Security의 `PasswordEncoder`를 사용하고 있다. 기본 구현으로는 BCrypt가 널리 쓰이지만, Argon2가 메모리 집약적인 설계 덕분에 GPU 기반 공격에 더 강하다는 점에서 보안 커뮤니티의 권장도가 높아졌다. 애플리케이션은 장기적으로도 안전한 비밀번호 저장 전략을 유지하고자 한다.

### 고려한 옵션

1. **BCryptPasswordEncoder**
   - 장점: Spring 기본값, 검증된 안정성, 설정 간단.
   - 단점: Argon2 대비 메모리 사용이 낮아 ASIC/GPU 공격에 상대적으로 취약.
2. **Argon2PasswordEncoder**
   - 장점: 메모리/연산/병렬도 파라미터를 조정해 공격 난이도 제어 가능, 최신 표준(RFC 9106).
   - 단점: 디폴트 파라미터에 대한 튜닝 필요, 메모리 요구량이 높은 환경에선 부담.
3. **Scrypt**
   - 장점: 메모리 하드, 오래된 표준화된 선택지.
   - 단점: Spring 제공 구현이 제한적, 운영/튜닝 경험 부족.

### 결정

`Argon2PasswordEncoder`를 채택한다.

### 근거

- GPU 및 ASIC에 대한 저항성을 고려할 때 장기적 보안성이 가장 높다고 판단.
- Spring Security 5 이상에서 `Argon2PasswordEncoder`가 기본 제공되어 도입 부담이 낮음.
- 설정을 통해 메모리/연산 비용을 환경에 맞게 조정할 수 있다.

### 후속 작업

- 운영 환경의 CPU/메모리 자원에 맞는 Argon2 파라미터 검토 및 벤치마크.
- 비밀번호 해싱 전략 변경 시 마이그레이션 정책 수립(기존 해시와의 호환, 순차 재해싱 등).
